          # Git 基本理解

          Git版本管理工具，有三大区域：

          1. 工作目录——存放项目代码的目录
          2. 暂存区——存放工作中更改的文件
          3. 代码仓库——当开发功能足够成为一个新版本时，提交到仓库，将暂存区中的代码复制一份存到代码仓库中

          # git 常用命令

          
  # 配置git用户名和密码
  git config --global user.name "your name"
  git config --global user.email "your email"

  # 查看当前配置
  git config --list

  # 初始化git仓库
  git init 

  # 查看当前仓库状态
  git status

  # 将工作目录中的文件添加到暂存区
  git add x.txt
  git add .
  git add a.txt b.txt

          如果有不想提交的文件夹或者文件，可以在项目下建立一个文件以.gitignore为后缀，写入不想提交的文件名即可。此时用git add .提交就是有选择的提交。

            # 将暂存区的代码提交到本地仓库，形成一个版本
  git commit -m "xxx"

  # 查看本地仓库中历史提交版本
  git log

  # 将暂存区中文件删除
  git rm --cashed filename
          1. 保证工作目录中的代码和暂存区一致
          2. 删除之后，工作目录任有此文件而暂存区没有，git将不做管理。
            # 用暂存区中的文件覆盖工作目录中的文件
  git checkout --filename

  # 回滚到本地仓库中特定版本并覆盖暂存区和工作目录
  git reset --hard commitID
          1. commitID可以git log查看，全部复制或者只取前六位
          2. 如果回滚到版本1，版本2若存在则会被自动删除

          # 分支相关命令

            # 查看分支 *代表当前分支
  git branch

  # 创建分支
  git branch branchname

  # 切换分支
  git checkout branchname

  # 创建并切换分支
  git checkout -b branchname

  # 删除分支
  git branch -D branchname

  # 合并分支 合并后来源分支任存在
  git merge branchname

          # github 相关命令

            # 为远程仓库创建别名
  git remote add origin git@github.com:<username>/xxx.git

  # 查看远程地址的详细信息
  git remote -v

  # 查看当前别名对应的远程仓库地址
  git remote show origin

  # 从远程仓库获取代码（拉取所有版本到本地）
  git clone origin

          加入到已有项目开发，需要先拉取所有版本到本地再进行开发

            # 从远程仓库拉取最新版本代码 远程 本地
  git pull origin master

  # 向远程仓库推送代码 git push <远程仓库> <本地分支名>:<远程分支名>
  git push origin master:master

  # 删除当前别名对应的远程仓库地址，解除关联
  git remote remove origin

          1. clone 和 pull 之间有什么区别？
